# Minecraft Mod Translator Client - Рефакторинг

## Структура проекта

После рефакторинга проект имеет следующую структуру:

```
client/
├── __init__.py                 # Пакет клиента
├── main.py                     # Главный файл запуска (CLI и GUI)
├── run_gui.py                  # Скрипт для запуска GUI
├── requirements.txt            # Зависимости
├── translator.py               # Оригинальный файл (сохранен для совместимости)
├── translator_client.py        # Рефакторенный основной клиент
├── README.md                   # Этот файл
└── gui/                        # Папка с GUI компонентами
    ├── __init__.py            # Пакет GUI
    └── translator_gui.py      # GUI интерфейс
```

## Новые возможности

1. **Графический интерфейс** - добавлен полнофункциональный GUI с использованием tkinter
2. **Разделение ответственности** - основная логика вынесена в отдельные модули
3. **Улучшенная структура** - код разделен на логические компоненты

## Запуск приложения

### CLI режим (консольный интерфейс)
```bash
python main.py [параметры]
```

Примеры:
```bash
# Обычная обработка
python main.py --input_dir ./mods --output_dir ./translated --threads 5

# Тестовый режим
python main.py --dry_run --input_dir ./mods

# Помощь
python main.py --help
```

### GUI режим (графический интерфейс)
```bash
python main.py --gui
```

Или с использованием скрипта:
```bash
python run_gui.py
```

## Поддерживаемые параметры CLI

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `--gui` | Запустить графический интерфейс | - |
| `--fb` | Использовать резервный переводчик | yes |
| `--cl` | Максимальное количество попыток | 3 |
| `--m` | Метод перевода | bing |
| `--f` | Исходный язык | en |
| `--t` | Целевой язык | ru |
| `--ai-provider` | AI провайдер | openrouter |
| `--input_dir` | Директория с JAR файлами | . |
| `--output_dir` | Директория для переведенных | 1 |
| `--output_invalid` | Директория для невалидных | 2 |
| `--output_corrupted` | Директория для поврежденных | 3 |
| `--threads` | Количество потоков | 3 |
| `--recursive` | Рекурсивный поиск | False |
| `--dry_run` | Тестовый режим | False |
| `--skip_health_check` | Пропустить проверку сервера | False |
| `--server_url` | URL API сервера | http://localhost:8250 |

## Функциональность GUI

Графический интерфейс предоставляет:

- Удобные элементы управления для всех параметров
- Кнопки для выбора директорий
- Комбинированные списки для выбора языков и провайдеров
- Панель логов в реальном времени
- Кнопки управления процессом
- Прогресс-бар (в будущих версиях)

## Зависимости

Основные зависимости:
- requests
- tqdm

Tkinter (графический интерфейс) входит в стандартную библиотеку Python, но в системах на базе Debian/Ubuntu может потребоваться установка пакета `python3-tk`.

## Совместимость

Оригинальный файл `translator.py` оставлен без изменений для обеспечения обратной совместимости. Новый функционал реализован в отдельных модулях.
